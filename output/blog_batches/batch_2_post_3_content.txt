# AWS Managed AI Services Complete Guide – Must-Know for AWS AI Practitioner (AIF-C01)

> 💡 **Exam Tip:** AIF-C01 frequently tests **scenario-based questions** like "Which AWS AI service solves this problem?" Map each service name to its core function in a 1:1 relationship.

---

## Why Use AWS Managed AI Services?

AWS Managed AI Services are **pre-trained ML services** designed to solve specific use cases without building or training models from scratch. You focus on **application logic**, not infrastructure.

| Advantage | Description |
|---|---|
| **Responsiveness & High Availability** | Available across multiple AWS Regions; responsive under heavy load |
| **Redundancy & Reliability** | Deployed across multiple AZs; survives single AZ failures |
| **High Performance** | Optimized CPUs, GPUs, and accelerators; better performance at lower cost than manual builds |
| **Token-Based Pricing** | Pay only per request or token; no over-provisioning needed |

> 💡 **Exam Tip:** When asked "Why use AWS Managed AI Services?" — remember: **pre-trained, no infrastructure, pay-per-use**.

## AWS Managed AI Services by Category

---

### 🤖 Generative AI

#### Amazon Bedrock
Fully managed service for accessing **Foundation Models (FMs)** and building generative AI applications.
- API access to models from Anthropic (Claude), Amazon (Titan), Mistral, and others
- Built-in fine-tuning, RAG, and agents
- No infrastructure management needed

#### Amazon Q Business / Q Developer
**Higher-level, purpose-built generative AI assistants** built on top of Bedrock — Q Business for employees using internal company data, Q Developer for AWS development and account management.

> 💡 **Exam Tip:** Bedrock = **FM platform**, Amazon Q = **purpose-specific AI assistant**. These operate at different abstraction levels.

---

### 📄 Text & Document Processing

#### Amazon Comprehend
**Natural Language Processing (NLP)** service that extracts meaning from text.
- Sentiment analysis, key phrase extraction
- Entity recognition (people, places, dates)
- Language detection, topic modeling

> 💡 **Exam Tip:** Extracting **"what does this text mean / what sentiment"** → Comprehend

#### Amazon Translate
**Neural machine translation** service supporting 75+ languages.

> 💡 **Exam Tip:** Converting text **into another language** → Translate

#### Amazon Textract
**Automatically extracts text and structured data** from documents (OCR and beyond).
- Recognizes tables, forms, and structured fields — not just plain text
- Used for medical records, tax documents, financial forms

> 💡 **Exam Tip:** Extracting data from **scanned documents, forms, tables** → Textract (don't confuse with Comprehend!)

---

### 👁️ Vision

#### Amazon Rekognition
**Image and video analysis** service.
- Facial recognition and comparison
- Object, scene, and activity detection
- Text in images, content moderation, celebrity recognition

> 💡 **Exam Tip:** Finding or identifying **things in images/videos** → Rekognition

---

### 🔍 Search

#### Amazon Kendra
**Intelligent enterprise search** powered by ML — answers natural language questions from internal documents.
- Connects to S3, SharePoint, Salesforce, and more
- Handles queries like "What is our travel expense policy?"

> 💡 **Exam Tip:** **Natural language search over internal documents** → Kendra

---

### 💬 Chatbots

#### Amazon Lex
**Conversational interface builder** (chatbots and voicebots) using the same technology as Amazon Alexa.
- Natural Language Understanding (NLU) + Automatic Speech Recognition (ASR)
- Integrates with Lambda for business logic

> 💡 **Exam Tip:** **Building a chatbot or voicebot** → Lex. Think: same tech as A**lex**a.

---

### 🔊 Speech

#### Amazon Polly
**Text-to-Speech (TTS)** — converts text into natural-sounding speech.
- Dozens of languages and voices
- Used in news readers, e-learning, accessibility tools

> 💡 **Exam Tip:** **Text → Speech** = Polly

#### Amazon Transcribe
**Speech-to-Text (STT)** — converts audio into text.
- Speaker diarization (identifies different speakers)
- Used for subtitles, call center analytics, medical transcription

> 💡 **Exam Tip:** **Speech → Text** = Transcribe

---

### 🎯 Recommendations

#### Amazon Personalize
**Real-time personalization and recommendation** service — Netflix-level recommendations without ML expertise.
- User behavior-driven personalization
- Product, content, and search personalization
- Real-time and batch recommendations

> 💡 **Exam Tip:** **"Recommended for you" features** → Personalize

---

### ⚙️ Full ML Platform

#### Amazon SageMaker
**Fully managed platform for the entire ML lifecycle** — Build, Train, Tune, Deploy, and Monitor.

> 💡 **Exam Tip:** SageMaker is fundamentally different from the services above. While others are **pre-trained APIs for specific tasks**, SageMaker is the platform where you **build your own ML models from scratch**.

## Service Selection Cheat Sheet

| Scenario | Answer |
|---|---|
| Detect faces/objects in images | **Rekognition** |
| Extract data from scanned forms/documents | **Textract** |
| Analyze text sentiment or key phrases | **Comprehend** |
| Translate text to another language | **Translate** |
| Natural language search over internal docs | **Kendra** |
| Build a chatbot or voicebot | **Lex** |
| Convert text to speech | **Polly** |
| Convert speech to text | **Transcribe** |
| Personalized product/content recommendations | **Personalize** |
| Access foundation models / build GenAI apps | **Bedrock** |
| Internal data-based AI assistant (employees) | **Amazon Q Business** |
| AWS dev / account management AI assistant | **Amazon Q Developer** |
| Build, train, and deploy custom ML models | **SageMaker** |

---

## Polly vs Transcribe vs Lex — The Confusing Trio

| Service | Direction | Memory Hook |
|---|---|---|
| **Polly** | Text → Speech | **P**olly **P**roduces speech |
| **Transcribe** | Speech → Text | **T**ranscribe = **T**ype what you hear |
| **Lex** | Understands conversation + builds chatbot | **L**ex = A**l**exa technology |

---

## Textract vs Comprehend

| Service | Input | Output | Purpose |
|---|---|---|---|
| **Textract** | Scanned images, PDFs | Text, tables, form data | **Extract** data from documents |
| **Comprehend** | Existing text | Sentiment, entities, key phrases | **Analyze** the meaning of text |
