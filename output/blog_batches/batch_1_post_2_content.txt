# Mastering AWS EC2

> 💡 **Exam Tip:** EC2 is one of the most heavily tested services on the CLF-C02 exam. Focus on instance types, purchasing options, and security groups!

---

## 1. What is Amazon EC2?

**Amazon EC2 (Elastic Compute Cloud)** lets you rent virtual servers in the cloud. You can scale compute capacity up or down based on demand, making it cost-effective for variable workloads.

### EC2 Building Blocks

| Component | Description |
|-----------|-------------|
| **AMI** | Template with OS + pre-configured software |
| **Instance Type** | Combination of CPU, memory, and network |
| **Storage** | EBS (persistent) or Instance Store (temporary) |
| **Security Group** | Virtual firewall at the instance level |
| **User Data** | Script auto-executed on first boot |

> 📝 **Exam Tip:** EC2 User Data runs **only once — on the very first start** of an instance!

## 2. EC2 Instance Types

AWS offers instance types optimized for different workloads. Know the naming convention:

> Example: `m5.xlarge` → **m** (type) + **5** (generation) + **xlarge** (size)

### Instance Types at a Glance

| Type | Characteristics | Key Use Cases |
|------|----------------|---------------|
| **General Purpose** | Balanced CPU, memory, network | Web servers, small DBs, dev/test |
| **Compute Optimized** | High-performance processors | Batch processing, ML inference, game servers |
| **Memory Optimized** | Large memory capacity | In-memory caches (Redis), high-performance DBs |
| **Storage Optimized** | High sequential I/O | NoSQL DBs, data warehousing, big data |

> 📝 **Exam Tips:**
> - `t2.micro` — **AWS Free Tier** (750 hours/month free)
> - `c` = Compute, `r` = RAM (Memory), `i` = I/O (Storage)

## 3. Security Groups

Security Groups act as a **virtual firewall** attached to EC2 instances, controlling inbound and outbound traffic.

### Default Behavior

| Direction | Default |
|-----------|---------|
| **Inbound** | **Deny all** (must explicitly allow) |
| **Outbound** | **Allow all** |

### Classic Ports to Memorize

| Port | Protocol | Purpose |
|------|----------|---------|
| **22** | TCP | SSH — remote Linux access |
| **80** | TCP | HTTP — web traffic |
| **443** | TCP | HTTPS — secure web traffic |
| **3389** | TCP | RDP — remote Windows access |
| **21** | TCP | FTP Control |
| **20** | TCP | FTP Data Transfer |

> 📝 **Exam Tips:**
> - Security Groups operate at the **instance level** (vs. Network ACLs at subnet level)
> - One instance can have **multiple security groups**
> - Security Groups support **Allow rules only** — no explicit deny

## 4. EC2 Purchasing Options — This Is Crucial!

Purchasing option comparison questions appear frequently on the CLF-C02 exam. Learn each option through real-world scenarios!

### Comparison Table

| Option | Discount | Commitment | Best For |
|--------|----------|------------|----------|
| **On-Demand** | None | None | Short-term, unpredictable workloads |
| **Reserved Instances** | Up to 72% | 1 or 3 years | Steady-state apps (e.g., databases) |
| **Savings Plans** | Up to 72% | 1 or 3 years | Flexible instance needs with usage commitment |
| **Spot Instances** | Up to 90% | None | Fault-tolerant batch jobs, data analysis |
| **Dedicated Hosts** | None | On-Demand / Reserved | Compliance, BYOL licensing |
| **Dedicated Instances** | None | None | Hardware isolation without full host |
| **Capacity Reservations** | None | None | Guaranteed capacity in a specific AZ |

### 🏨 Hotel Analogy

| Option | Analogy |
|--------|---------|
| **On-Demand** | Walk in anytime and pay full price |
| **Reserved** | Book 1–3 years in advance for a big discount |
| **Spot** | Bid on empty rooms — cheapest but can be kicked out anytime |
| **Dedicated Host** | Rent an entire wing of the hotel |
| **Dedicated Instance** | Private room, but share common areas |
| **Capacity Reservation** | Hold a room regardless of whether you use it |

> 📝 **Exam Tips:**
> - **Cheapest:** Spot Instances (up to 90% off, but can be interrupted!)
> - **Never use Spot for critical DBs or production workloads**
> - **BYOL / compliance:** Use Dedicated Hosts
> - **Most expensive:** Dedicated Hosts

### Price Comparison (m4.large, us-east-1)

| Option | Hourly | ~Monthly |

| Savings Plans (1yr) | $0.058 | $41.76 |

## 5. Shared Responsibility Model for EC2

| AWS Responsibility | Customer Responsibility |
|-------------------|------------------------|
| Physical server & data center security | OS patching and updates |
| Network infrastructure & DDoS protection | Security group & VPC configuration |
| Hypervisor security | Application-level security |
| Infrastructure compliance | Data encryption management |

> 📝 **Exam Tip:** **OS patches, security group rules, and application security are always the customer's responsibility!**

---

## 6. EC2 Quick Summary

- **AMI:** Base image for an instance (OS + software)
- **User Data:** Automation script that runs once on first boot
- **Security Group:** Instance-level virtual firewall (inbound denied by default)
- **SSH (Port 22):** Remote access for Linux EC2
- **RDP (Port 3389):** Remote access for Windows EC2
- **t2.micro:** Free Tier eligible instance
