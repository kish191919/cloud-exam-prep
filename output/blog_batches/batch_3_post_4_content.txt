## 🌩️ Domain 1: Cloud Concepts

### 6 Key Benefits of AWS Cloud (Memorize these!)

- **Security** — Shared model protects physical + logical layers
- **Reliability** — Auto-recovery, multi-AZ design
- **High Availability** — Minimize downtime
- **Elasticity** — Scale in/out automatically with demand
- **Agility** — Fast experimentation and deployment
- **Pay-as-you-go** — Only pay for what you use

### TCO (Total Cost of Ownership)

| Type | Description |
|------|-------------|
| **CapEx** | Upfront investments (servers, data centers) |
| **OpEx** | Ongoing usage costs (cloud services) |

> 🎯 **Exam Tip:** Moving to the cloud shifts costs from CapEx to OpEx — this is how AWS reduces TCO!

### Cloud Architecture Design Principles

- **Design for Failure** — Assume anything can fail
- **Decouple Components** — Avoid monolithic architecture
- **Implement Elasticity** — Build systems that scale
- **Think Parallel** — Use parallel processing for speed

## 🔐 Domain 2: Security & Compliance

### Shared Responsibility Model — Most Important Topic!
```
AWS Responsibility: Security OF the Cloud
  → Physical infrastructure, hardware, global network

Customer Responsibility: Security IN the Cloud
  → Data, OS patching, application code, IAM configuration
```

> 🎯 **Exam Tip:** Responsibility shifts based on service type!
> - **EC2** → Customer manages OS and applications
> - **RDS** → AWS manages OS/DB engine, customer manages data
> - **Lambda** → Customer only manages code

### AWS IAM Essentials

| Concept | Description |
|---------|-------------|
| **User** | Individual account with access keys/password |
| **Group** | Apply policies to multiple users at once |
| **Role** | Grant temporary permissions to services/users |
| **Policy** | JSON document defining permissions |
| **MFA** | Must enable on root account! |

> 🎯 **Exam Tip:** Always apply the **Principle of Least Privilege** — grant only the permissions needed!

### Key Security & Audit Services

- **Amazon CloudWatch** — Monitoring and alarms
- **AWS CloudTrail** — Logs who did what and when (API calls)
- **AWS Config** — Tracks resource configuration changes

## 🛠️ Domain 3: Technology

### AWS Global Infrastructure
```
Region
  └── Availability Zone (AZ) × 2–6
        └── Data Centers
Edge Location — CloudFront, Global Accelerator
```

> 🎯 **Exam Tips:**
> - **Multiple AZs** → High Availability
> - **Multiple Regions** → Disaster Recovery, data sovereignty, low latency
> - **Edge Locations** → Content caching, reduced latency

### Core Compute Services

| Service | Key Feature |
|---------|-------------|
| **EC2** | Virtual servers, you manage the OS |
| **Lambda** | Serverless, event-driven code execution |
| **ECS / EKS** | Container orchestration |
| **Auto Scaling** | Automatic capacity adjustment |
| **ELB** | Distributes incoming traffic |

### Core Storage Services

| Service | Key Feature |
|---------|-------------|
| **S3** | Object storage, unlimited capacity |
| **EBS** | Block storage attached to EC2 |
| **EFS** | Shared file storage for multiple EC2s |
| **S3 Glacier** | Low-cost long-term archival |
| **Snowball** | Physical device for large-scale data migration |

### Core Database Services

| Service | Key Feature |
|---------|-------------|
| **RDS** | Managed relational DB (MySQL, PostgreSQL, etc.) |
| **DynamoDB** | Serverless NoSQL DB |
| **Redshift** | Data warehouse for analytics |

## 💰 Domain 4: Billing & Pricing

### EC2 Pricing Models

| Type | Best For | Savings |
|------|----------|---------|
| **On-Demand** | Unpredictable, short-term workloads | Baseline |
| **Reserved** | Stable 1–3 year workloads | Up to 72% |
| **Spot** | Interruptible batch workloads | Up to 90% |

> 🎯 **Exam Tip:** Spot Instances can be reclaimed by AWS with a 2-minute warning when capacity is needed!

### AWS Organizations & Consolidated Billing

- **Consolidated Billing** — Combine costs from multiple accounts
- Share volume discounts across accounts
- Allocate costs per department using tags

### Cost Management Tools

| Tool | Function |
|------|----------|
| **Cost Explorer** | Visualize and forecast costs |
| **AWS Budgets** | Set alerts for budget thresholds |
| **Cost & Usage Report** | Detailed usage breakdown |
| **Tags** | Track costs by resource/project |

### AWS Trusted Advisor

Provides recommendations across 5 categories:
**Cost Optimization / Performance / Security / Fault Tolerance / Service Limits**
