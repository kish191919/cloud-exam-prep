# 품질 자기검증 체크리스트 (STEP 5)

재설계된 문제를 SQL로 생성하기 전에 다음 8개 항목을 순서대로 검증한다.
**하나라도 실패하면 재설계를 재시도한다 (최대 2회).**

---

## 체크리스트 항목

### ✅ 항목 1: 정답 논리 유효성
- [ ] 정답 보기(`correct_option_id`)가 명확하게 옳은가?
- [ ] 정답을 뒷받침하는 AWS 공식 기능·특성이 존재하는가?
- [ ] 틀린 정보나 허구의 AWS 기능을 정답으로 제시하지 않았는가?

**판정 기준:** "왜 이 보기가 정답인가?"를 한 문장으로 명확히 설명할 수 있어야 함.

---

### ✅ 항목 2: 오답 그럴듯함
- [ ] 각 오답이 수험생이 실제로 고민할 만큼 그럴듯한가?
- [ ] 오답이 완전히 무관하거나 명백히 틀린 내용은 아닌가?
- [ ] 오답 3개가 각각 서로 다른 개념·서비스를 다루는가?

**판정 기준:** 각 오답에 대해 "왜 이 보기가 정답처럼 보일 수 있는가?"를 설명할 수 있어야 함.

---

### ✅ 항목 3: 한국어 자연스러움
- [ ] 번역투 표현(직역체, 어색한 어순)이 없는가?
- [ ] 조사, 어미, 경어체가 일관되게 사용되었는가?
- [ ] 한국어 수험 문체에 맞는 표현인가?

**판정 기준:** 원어민 한국어 화자가 어색함을 느끼지 않아야 함.

---

### ✅ 항목 4: 원문 AWS 서비스명 보존
- [ ] 원문에 등장한 AWS 서비스명이 재설계 후에도 동일하게 사용되는가?
- [ ] 서비스명을 번역·축약·대체하지 않았는가?

**판정 기준:** 원문의 모든 AWS 서비스명이 재설계된 문제·보기·해설에 원문 그대로 등장해야 함.

---

### ✅ 항목 5: 보기 길이 유사성 및 형식 단순성
- [ ] 정답 보기만 유독 길거나 짧아서 힌트가 되지 않는가?
- [ ] 4개 보기의 텍스트 길이가 대략 비슷한가? (최대 50% 차이 이내)
- [ ] 보기가 서비스명·간결한 명사구 형식인가? (40자 이내 권장)
- [ ] 파이프(`|`) 구분 복합 매핑 형식을 사용하지 않았는가?
- [ ] 한 보기 안에 3개 이상의 서비스를 열거하지 않았는가?

**판정 기준:** 보기 4개를 스크롤 없이 한눈에 읽고 비교할 수 있어야 함.
좋은 형식: `Amazon Textract` / `Amazon Transcribe + Amazon Comprehend`
나쁜 형식: `요구 1: SageMaker A | 요구 2: SageMaker B | 요구 3: ...`

---

### ✅ 항목 6: 항목 간 쏠림 없음
- [ ] 두 보기가 거의 같은 내용을 다르게 표현하지 않았는가?
- [ ] 의미상 중복되는 보기 쌍이 없는가?

**판정 기준:** 4개 보기가 각각 서로 다른 개념이나 접근법을 나타내야 함.

---

### ✅ 항목 7: 두 번 읽으면 답이 명확
- [ ] 문제를 두 번 읽었을 때 정답이 일관되게 동일한가?
- [ ] 모호하거나 여러 해석이 가능한 질문이 아닌가?

**판정 기준:** 질문이 명확하고 정답이 하나뿐임을 확인할 수 있어야 함.

---

### ✅ 항목 8: 단일 개념 집중 (문제 범위)
- [ ] 한 문제가 단 하나의 핵심 판단을 테스트하는가?
- [ ] 여러 요구사항을 여러 서비스에 동시 매핑하는 구조가 아닌가?
- [ ] 보기 4개가 동일한 판단 기준(같은 종류의 선택지)으로 비교되는가?

**판정 기준:** 문제의 핵심 질문을 한 문장으로 요약할 수 있어야 함.
좋은 예: "음성을 텍스트로 변환하고 감정을 분석하는 서비스 조합은?"
나쁜 예: "4가지 요구사항에 각각 맞는 서비스를 올바르게 연결한 것은?" (4가지 판단 동시 수행)

---

## 검증 실패 시 처리

| 실패 항목 | 재시도 방향 |
|----------|------------|
| 항목 1 (정답 논리) | 정답 보기와 해설을 다시 작성 |
| 항목 2 (오답 그럴듯함) | 오답 보기 1~2개를 더 그럴듯하게 교체 |
| 항목 3 (한국어) | 해당 문장 표현 수정 |
| 항목 4 (서비스명) | 서비스명 원복 |
| 항목 5 (보기 길이·형식) | 파이프 구분 보기 → 서비스명 단독 or `A + B` 조합으로 재작성 |
| 항목 6 (중복) | 중복 보기 중 하나를 다른 개념으로 교체 |
| 항목 7 (모호함) | 질문·시나리오 재작성으로 명확성 확보 |
| 항목 8 (단일 개념) | 4가지 매핑 구조 → 핵심 요구사항 1개로 범위 축소 후 재설계 |

**최대 재시도 횟수: 2회**
