# 품질 검증 체크리스트 (에이전트 전용 — 간결 버전)

STEP 5에서 아래 10개 항목을 순서대로 확인한다. 1~9번 FAIL이면 재시도(최대 2회). 10번 FAIL이면 재시도 없이 즉시 에스컬레이션.

---

1. **정답 논리 유효성** — 정답이 명확히 옳고, AWS 공식 기능을 근거로 하는가?
2. **오답 그럴듯함** — 오답 3개가 각각 다른 개념을 다루며 수험생이 고민할 만한가?
3. **한국어 자연스러움** — 번역투 없이 수험 문체(-입니다/-합니다)가 일관된가?
4. **AWS 서비스명 보존** — 원문 서비스명을 번역·축약 없이 그대로 사용했는가?
5. **보기 길이·형식** — 40자 이내, 4개 길이 균일, 파이프(`|`) 구분 금지, 서비스 3개+ 나열 금지?
6. **보기 중복 없음** — 4개 보기가 각각 서로 다른 서비스·접근법인가?
7. **질문 명확성** — 모호함 없이 정답이 단 하나인가?
8. **단일 개념 집중** — 하나의 핵심 판단만 테스트하는가? (다중 매핑 구조 아닌가?)
9. **보기 순서 무작위화 (규칙 12)** — 재설계 `correct_option_id`가 원문 `answer` 위치와 다른가?
10. **원본 정답 AWS 기술적 유효성** — `correct_answer_concept`이 원문 질문과 보기 전체를 검토했을 때 다른 보기들 대비 명확히 우월한 정답인가? AWS 공식 기능을 근거로 판단. ⚠️ FAIL 시 재시도 없이 즉시 에스컬레이션.

---

결과 출력 형식 (각 항목 한 줄씩):
```
[PASS] 정답 논리 유효성
[PASS] 오답 그럴듯함
[PASS] 한국어 자연스러움
[PASS] 원문 서비스명 보존
[PASS] 보기 길이·형식
[PASS] 보기 중복 없음
[PASS] 질문 명확성
[PASS] 단일 개념 집중
[PASS] 규칙 12: 정답 위치 원문과 다름
[PASS] 원본 정답 AWS 기술적 유효성
```
