AWS Certified AI Practitioner(AIF-C01) 시험은 특정 직무와 관계없이 AI/ML, 생성형 AI 기술 및 관련 AWS 서비스와 도구에 대한 전반적인 지식을 효과적으로 입증할 수 있는 개인을 대상으로 합니다.

이 시험은 또한 응시자가 다음과 같은 과제를 수행할 수 있는 능력을 검증합니다.

일반적인 AI, 머신러닝, 생성형 AI의 개념, 방법, 전략을 AWS 환경에서 이해합니다.
지원자는 소속 조직 내에서 관련성 있는 질문을 하기 위해 AI/ML 및 생성형 AI 기술의 적절한 사용법을 이해해야 합니다.
특정 사용 사례에 적용할 적절한 AI/ML 기술 유형을 결정합니다.
인공지능(AI), 머신러닝(ML), 생성형 AI 기술을 책임감 있게 사용하십시오.
대상 후보자 설명
해당 지원자는 AWS 기반 AI/ML 기술 경험이 최대 6개월 이상이어야 합니다. AWS에서 AI/ML 솔루션을 사용해본 경험이 있으면 좋지만, 반드시 직접 구축해본 경험이 있을 필요는 없습니다.

권장되는 AWS 지식
지원자는 다음과 같은 AWS 지식을 갖추어야 합니다.

핵심 AWS 서비스(예: Amazon EC2, Amazon S3, AWS Lambda, Amazon SageMaker) 및 AWS 핵심 서비스 사용 사례에 대한 이해
AWS 클라우드 환경에서 보안 및 규정 준수에 대한 AWS 공동 책임 모델에 대한 이해
AWS 리소스에 대한 접근을 보호하고 제어하기 위한 AWS Identity and Access Management(IAM)에 대한 이해
AWS 리전, 가용 영역 및 엣지 로케이션 개념을 포함한 AWS 글로벌 인프라에 대한 이해
AWS 서비스 가격 모델에 대한 이해
시험 내용
질문 유형
시험에는 다음 유형의 문제가 하나 이상 포함됩니다.

객관식 문제 : 정답 1개와 오답(선택지) 3개가 있습니다.
복수 선택형 : 다섯 개 이상의 보기 중에서 두 개 이상의 정답을 선택해야 합니다. 모든 보기를 정답으로 선택해야 정답으로 인정됩니다.
순서 배열 : 주어진 과제를 완료하기 위해 3~5개의 보기 중에서 정답을 선택해야 합니다. 정답을 선택하고 올바른 순서대로 배열해야 문제를 풀고 점수를 받을 수 있습니다.
매칭형 문제 : 3~7개의 질문과 답변 목록이 제시됩니다. 모든 쌍을 정확하게 매칭해야 정답을 맞출 수 있습니다.
사례 연구 : 하나의 시나리오와 해당 시나리오에 대한 두 개 이상의 질문으로 구성됩니다. 시나리오는 사례 연구의 각 질문에 대해 동일합니다. 사례 연구의 각 질문은 개별적으로 평가됩니다. 사례 연구에서 정답을 맞춘 각 질문에 대해 점수를 받게 됩니다.
답하지 않은 문제는 오답으로 처리되며, 찍기에 대한 감점은 없습니다. 시험은 총 50문항 으로 구성되어 있으며, 각 문항 이 점수에 영향을 미칩니다.

점수가 매겨지지 않은 콘텐츠
시험에는 점수에 영향을 미치지 않는 15개의 문항이 포함되어 있습니다. AWS는 이러한 문항에 대한 성능 정보를 수집하여 향후 점수 산정 문항으로 활용할지 여부를 평가합니다. 이러한 문항은 시험에서 별도로 표시되지 않습니다.

시험 결과
AWS Certified AI Practitioner(AIF-C01) 시험은 합격 또는 불합격으로 판정됩니다 . 시험 점수는 인증 업계의 모범 사례 및 지침을 따르는 AWS 전문가들이 정한 최소 기준에 따라 산정됩니다.
시험 결과는 100점에서 1,000점 사이의 환산 점수로 발표됩니다. 최소 합격 점수는 700점입니다 . 점수는 시험 전반에 걸친 성적과 합격 여부를 보여줍니다. 환산 점수 제도는 난이도가 다소 다를 수 있는 여러 시험 유형 간의 점수 균등화를 위해 사용됩니다.
성적표에는 각 영역별 성적 등급표가 포함될 수 있습니다. 이 시험은 보상 채점 방식을 사용하므로 각 영역에서 합격점을 받을 필요는 없습니다. 전체 시험만 통과하면 됩니다.
시험의 각 영역은 특정 가중치를 가지고 있으므로, 일부 영역에는 다른 영역보다 더 많은 문제가 출제됩니다. 분류표에는 여러분의 강점과 약점을 파악하는 데 도움이 되는 일반적인 정보가 담겨 있습니다. 영역별 피드백을 해석할 때는 주의를 기울이세요.
콘텐츠 개요
이 시험 안내서에는 시험의 배점, 내용 영역 및 과제 설명이 포함되어 있습니다. 이 안내서는 시험에 포함되는 모든 내용을 망라하는 것은 아닙니다. 하지만 각 과제 설명에 대한 추가적인 맥락 정보를 제공하여 시험 준비에 도움을 드립니다.

시험은 다음과 같은 내용 영역과 가중치를 갖습니다.

영역 1 : 인공지능 및 머신러닝 기초 (평가 항목의 20%)
영역 2 : 생성형 인공지능의 기초 (채점 내용의 24%)
영역 3 : 기초 모델의 응용 (평가 항목의 28%)
영역 4 : 책임감 있는 AI를 위한 가이드라인 (평가 항목의 14%)
영역 5 : AI 솔루션의 보안, 규정 준수 및 거버넌스 (평가 대상 콘텐츠의 14%)
영역 1: 인공지능 및 머신러닝의 기초
인공지능의 기본 개념과 용어를 설명하십시오.
인공지능의 기본 용어(예: AI, ML, 딥러닝, 신경망, 컴퓨터 비전, 자연어 처리[NLP], 모델, 알고리즘, 학습 및 추론, 편향, 공정성, 적합성, 대규모 언어 모델[LLM])를 정의하십시오.
AI, ML, 딥러닝의 유사점과 차이점을 설명하세요.
다양한 유형의 추론 방식(예: 배치 추론, 실시간 추론)에 대해 설명하십시오.
AI 모델에서 사용되는 다양한 데이터 유형(예: 레이블이 지정된 데이터와 지정되지 않은 데이터, 표 형식 데이터, 시계열 데이터, 이미지, 텍스트, 정형 데이터와 비정형 데이터)에 대해 설명하십시오.
지도 학습, 비지도 학습, 강화 학습에 대해 설명하십시오.
AI의 실질적인 활용 사례를 파악하세요
AI/ML이 가치를 제공할 수 있는 애플리케이션(예: 인간의 의사 결정 지원, 솔루션 확장성, 자동화)을 파악하십시오.
AI/ML 솔루션이 적합하지 않은 경우를 파악하십시오(예: 비용 편익 분석, 예측 대신 특정 결과가 필요한 상황).
특정 사용 사례에 적합한 머신러닝 기법을 선택하십시오(예: 회귀 분석, 분류, 클러스터링).
실제 인공지능 응용 사례를 제시하십시오(예: 컴퓨터 비전, 자연어 처리, 음성 인식, 추천 시스템, 사기 탐지, 예측).
AWS 관리형 AI/ML 서비스(예: SageMaker, Amazon Transcribe, Amazon Translate, Amazon Comprehend, Amazon Lex, Amazon Polly)의 기능을 설명하십시오.
머신러닝 개발 수명주기를 설명하세요.
머신러닝 파이프라인의 구성 요소를 설명하십시오(예: 데이터 수집, 탐색적 데이터 분석[EDA], 데이터 전처리, 특징 엔지니어링, 모델 학습, 하이퍼파라미터 튜닝, 평가, 배포, 모니터링).
머신러닝 모델의 출처를 이해합니다(예: 오픈 소스 사전 학습 모델, 사용자 지정 모델 학습).
실제 운영 환경에서 모델을 사용하는 방법을 설명하세요(예: 관리형 API 서비스, 자체 호스팅 API).
머신러닝 파이프라인의 각 단계에 적합한 AWS 서비스 및 기능을 파악합니다(예: SageMaker, Amazon SageMaker Data Wrangler, Amazon SageMaker Feature Store, Amazon SageMaker Model Monitor).
머신러닝 운영(MLOps)의 기본 개념(예: 실험, 반복 가능한 프로세스, 확장 가능한 시스템, 기술 부채 관리, 프로덕션 준비 완료, 모델 모니터링, 모델 재학습)을 이해합니다.
머신러닝 모델을 평가하기 위해 모델 성능 지표(예: 정확도, ROC 곡선 아래 면적[AUC], F1 점수)와 비즈니스 지표(예: 사용자당 비용, 개발 비용, 고객 피드백, 투자 수익률[ROI])를 이해해야 합니다.
영역 2: 생성형 인공지능의 기초
생성형 인공지능의 기본 개념을 설명하세요.
생성형 AI의 기본 개념(예: 토큰, 청킹, 임베딩, 벡터, 프롬프트 엔지니어링, 트랜스포머 기반 LLM, 파운데이션 모델, 멀티모달 모델, 확산 모델)을 이해합니다.
생성형 AI 모델의 잠재적 활용 사례를 파악합니다(예: 이미지, 비디오 및 오디오 생성, 요약, 챗봇, 번역, 코드 생성, 고객 서비스 상담원, 검색, 추천 엔진).
기초 모델의 수명 주기(예: 데이터 선택, 모델 선택, 사전 학습, 미세 조정, 평가, 배포, 피드백)를 설명하십시오.
비즈니스 문제 해결을 위한 생성형 인공지능의 기능과 한계를 이해하십시오.
생성형 인공지능의 장점(예: 적응성, 반응성, 단순성)을 설명하십시오.
생성형 AI 솔루션의 단점(예: 환각, 해석 가능성, 부정확성, 비결정성)을 파악하십시오.
적절한 생성형 AI 모델을 선택하기 위해 고려해야 할 다양한 요소(예: 모델 유형, 성능 요구 사항, 기능, 제약 조건, 규정 준수)를 이해해야 합니다.
생성형 AI 애플리케이션의 비즈니스 가치 및 지표를 결정합니다(예: 도메인 간 성능, 효율성, 전환율, 사용자당 평균 수익, 정확도, 고객 생애 가치).
생성형 AI 애플리케이션 구축을 위한 AWS 인프라 및 기술에 대해 설명하십시오.
생성형 AI 애플리케이션 개발에 필요한 AWS 서비스 및 기능을 파악합니다(예: Amazon SageMaker JumpStart, Amazon Bedrock, Amazon Bedrock Playground인 PartyRock, Amazon Q).
AWS 생성형 AI 서비스를 사용하여 애플리케이션을 구축할 때의 장점(예: 접근성, 진입 장벽 낮음, 효율성, 비용 효율성, 시장 출시 속도, 비즈니스 목표 달성 능력)을 설명하십시오.
생성형 AI 애플리케이션을 위한 AWS 인프라의 이점(예: 보안, 규정 준수, 책임성, 안전성)을 이해하십시오.
AWS 생성형 AI 서비스의 비용 관련 절충점을 이해하십시오(예: 응답성, 가용성, 이중화, 성능, 지역 범위, 토큰 기반 가격 책정, 프로비저닝 처리량, 사용자 지정 모델).
영역 3: 기초 모델의 응용
기초 모델을 사용하는 애플리케이션의 설계 고려 사항에 대해 설명하십시오.
사전 학습된 모델을 선택하기 위한 선정 기준을 파악합니다(예: 비용, 모달리티, 지연 시간, 다국어 지원, 모델 크기, 모델 복잡성, 맞춤 설정, 입력/출력 길이).
추론 매개변수가 모델 응답에 미치는 영향(예: 온도, 입력/출력 길이)을 이해합니다.
검색 증강 생성(RAG)을 정의하고 비즈니스 응용 분야(예: Amazon Bedrock, 지식 기반)에 대해 설명하십시오.
벡터 데이터베이스 내에 임베딩을 저장하는 데 도움이 되는 AWS 서비스를 식별하십시오(예: Amazon OpenSearch Service, Amazon Aurora, Amazon Neptune, Amazon DocumentDB[MongoDB 호환성 포함], Amazon RDS for PostgreSQL).
기초 모델 맞춤화에 대한 다양한 접근 방식(예: 사전 학습, 미세 조정, 컨텍스트 학습, RAG)의 비용 대비 효과를 설명하십시오.
다단계 작업에서 에이전트의 역할을 이해합니다(예: Amazon Bedrock용 에이전트).
효과적인 신속 엔지니어링 기술을 선택하십시오.
프롬프트 엔지니어링의 개념과 구성 요소(예: 맥락, 지시, 부정적 프롬프트, 잠재 공간 모델)를 설명하십시오.
신속한 엔지니어링 기법(예: 사고 연쇄, 제로샷, 싱글샷, 피샷, 프롬프트 템플릿)을 이해하십시오.
신속한 엔지니어링을 위한 이점과 모범 사례(예: 응답 품질 개선, 실험, 가이드라인, 탐색, 구체성 및 간결성, 다양한 의견 활용)를 이해하십시오.
신속한 구조 공학의 잠재적 위험과 한계를 정의하십시오(예: 노출, 중독, 납치, 탈옥).
기초 모델의 학습 및 미세 조정 과정을 설명하십시오.
기초 모델 학습의 핵심 요소(예: 사전 학습, 미세 조정, 지속적인 사전 학습)를 설명하십시오.
기초 모델을 미세 조정하는 방법을 정의합니다(예: 명령어 조정, 특정 도메인에 맞춘 모델 적용, 전이 학습, 지속적인 사전 훈련).
기초 모델을 미세 조정하기 위한 데이터 준비 방법(예: 데이터 큐레이션, 거버넌스, 크기, 레이블링, 대표성, 인간 피드백 기반 강화 학습[RLHF])을 설명하십시오.
