1) Guardrails for Amazon Bedrock는 생성된 이미지가 AI로 만들어졌는지 판별하는 기능이며, 워터마크 감지 기능은 유해하거나 부적절한 프롬프트를 차단하는 기능이다.
2) Guardrails for Amazon Bedrock는 모델 학습 데이터에서 부적절한 데이터를 제거하는 기능이고, 워터마크 감지 기능은 학습 데이터를 자동으로 라벨링하는 기능이다.
3) Guardrails for Amazon Bedrock는 모델의 비용 최적화와 지연 시간 감소를 위한 성능 튜닝 기능이고, 워터마크 감지 기능은 모델 엔드포인트의 오토스케일링을 수행하는 기능이다.
4) Guardrails for Amazon Bedrock는 모델 출력(텍스트/이미지)이 정책을 위반하지 않도록 사전에 필터링·차단하는 안전 제어 기능이고, 워터마크 감지 기능은 생성된 이미지에 포함된 보이지 않는 워터마크를 탐지해 해당 이미지가 특정 생성형 모델로 만들어졌는지 “사후에” 식별하는 기능이다.

4. 한 뉴스 큐레이션 플랫폼이 기사를 자동으로 분류하는 ML 모델을 개발하고 있습니다. 기사 하나는 주제 카테고리를 하나만 가질 수도 있지만, 어떤 기사는 여러 속성을 동시에 가질 수 있습니다. 예를 들어 같은 기사가 '정치'이면서 '경제'이고, '국제' 태그도 함께 붙을 수 있습니다. 다음 중 Multi-class 분류와 Multi-label 분류의 차이를 가장 올바르게 설명한 것은 무엇인가요?

1) Multi-class 분류는 한 샘플이 여러 라벨을 동시에 가질 수 있고, Multi-label 분류는 한 샘플이 단 하나의 라벨만 가진다.
2) Multi-class 분류는 각 데이터가 하나의 클래스에만 속하도록 분류하고, Multi-label 분류는 각 데이터가 여러 클래스에 동시에 속할 수 있도록 분류한다
3) Multi-class 분류는 연속값(회귀)을 예측하고, Multi-label 분류는 범주값(분류)을 예측한다.
4) Multi-class 분류는 비지도 학습이고, Multi-label 분류는 지도 학습이다.
