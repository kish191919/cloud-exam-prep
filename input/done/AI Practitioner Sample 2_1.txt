- 학습 데이터에서는 오차가 매우 낮지만 검증 데이터에서는 오차가 크게 증가
- 학습 데이터와 검증 데이터 모두에서 오차가 높게 유지
팀은 두 실패 패턴의 개념적 차이를 정확히 이해하고 각각에 맞는 해결책을 적용하려 합니다. 다음 중 머신러닝에서 overfitting과 underfitting을 가장 정확하게 구분한 설명은 무엇입니까?

1) Overfitting은 학습/검증 데이터 모두에서 오차가 높고, Underfitting은 학습 데이터에서만 오차가 높아지는 상태다.

3) Overfitting은 데이터가 많을수록 반드시 발생하고, Underfitting은 데이터가 적을수록 반드시 발생한다.
4) Overfitting은 회귀(regression)에서만 나타나고, Underfitting은 분류(classification)에서만 나타난다.

- 분석 처리 과정 중 유전자 데이터가 변경되거나 손상되지 않았음을 보장
팀은 두 요건이 각각 어떤 보안 개념에 해당하는지 명확히 이해하고 설계에 반영하려 합니다. 다음 중 데이터 접근 제어와 데이터 무결성의 차이를 가장 정확하게 설명한 것은 무엇입니까?

1) 데이터 접근 제어는 사용자 인증과 권한 부여를 통해 누가 데이터에 접근할 수 있는지를 관리하고, 데이터 무결성은 데이터가 정확하고 일관되며 의도치 않게 변경되지 않았음을 보장한다.
2) 접근 제어는 데이터가 암호화되어 외부에서 읽지 못하게 만드는 것이고, 무결성은 장애가 나도 데이터가 계속 사용 가능하도록 이중화하는 것이다.
3) 접근 제어는 서비스가 중단되지 않도록 가용성을 높이는 것이고, 무결성은 특정 사용자가 데이터에 접근할 수 있는지 권한을 확인하는 것이다.
4) 접근 제어는 로그를 수집해 누가 접근했는지 추적하는 것이고, 무결성은 데이터의 기밀성을 유지해 노출을 방지하는 것이다.

14. 한 글로벌 이커머스 기업이 개인화 추천 문구 생성, 고객 리뷰 자동 요약, 실시간 CS 챗봇을 위해 Amazon Bedrock 도입을 검토하고 있습니다. 기술팀은 파운데이션 모델을 얼마나 유연하게 사용할 수 있는지, 모델 크기에 따른 비용 구조가 어떻게 되는지, 그리고 도메인 특화 파인튜닝 모델을 어떤 방식으로 운영할 수 있는지를 명확히 파악하려 합니다. 다음 중 Amazon Bedrock의 특징과 사용 방식에 대해 올바른 것을 선택하세요

1) Amazon Bedrock은 단일 AWS 전용 파운데이션 모델만 제공하며, 모델 크기와 무관하게 월 고정 구독료로만 과금된다.
2) 파인튜닝이나 커스터마이즈를 거친 모델은 안정적인 처리량 보장을 위해 Provisioned Throughput 모드에서만 사용할 수 있다.
3) Amazon Bedrock에서 도메인 특화 모델을 만들려면 반드시 고객이 GPU 인스턴스를 직접 프로비저닝하고 학습/배포 파이프라인 전체를 직접 운영해야 한다
4) Amazon Bedrock의 파인튜닝 모델은 로컬 환경에만 다운로드해 온프레미스에서만 운영할 수 있으며, AWS 내에서 호스팅/엔드포인트 형태로 운영할 수 없다.
